# Repository Guidelines

## 目的
この `AGENTS.md` は開発者向けドキュメントです。作業方針、変更履歴の要約、構成メモ、今後の課題、運用ルールなどを自由に記載して構いません。リポジトリの実態に合わせて更新し、内容を古くしないことを優先してください。

## 基本方針
- このリポジトリに関する説明文書は原則として日本語で記述します。
- エージェントや開発者からの報告、補足、メモも日本語で残します。
- 存在しないビルド手順やテスト手順を推測で書かないでください。
- 変更内容は短く具体的に書き、後から読んでも判断できる粒度を保ってください。

## リポジトリ構成
現在のリポジトリは Raspberry Pi 4B を balenaCloud で管理する監視基盤コンテナ群の管理用です。
- ルート: `docker-compose.yml`、`README.md`、運用ガイド類を配置
- `.github/workflows/`: GitHub Actions による自動デプロイ設定を配置
- `docs/`: 構成図や補助資料を配置

現時点の主対象サービスは `mariadb`、`zabbix-server`、`zabbix-web`、`grafana`、`cloudflared` です。fleet 名は `Pi-Smarthome`、device type は `Raspberry Pi 4 (using 64bit OS)` を前提とします。構成を変更した場合は、この文書と `README.md` の両方を更新してください。

## ドキュメント運用
- 新しいファイルは目的が分かる名前にします。例: `docs/setup-guide.md`
- Markdown の見出しは簡潔にし、本文は短い文で直接的に書きます。
- 参考コマンドやパスを載せる場合は、現時点の構成と一致していることを確認します。
- 作業メモを書く場合も、不要になった内容は整理してください。
- Balena Cloud の fleet 名、シークレット、公開ポートなどの運用前提が変わった場合は、README の手順も同時に修正してください。
- SD 消耗対策として、永続データは USB ストレージ前提で考えます。Compose だけで表現できない Host 側運用が絡む場合は、その制約を README と補助ドキュメントへ明記してください。

## README.md の更新ルール
- 各編集作業の後には、`README.md` を適切に作成または更新することを基本ルールとします。
- `README.md` には少なくとも、リポジトリの目的、現在の構成、主要ファイルやディレクトリの説明を反映してください。
- 構成変更、運用ルール変更、新規ドキュメント追加があった場合は、対応する説明を `README.md` に追記または修正してください。

## テストと確認
現時点では自動テスト環境はありません。変更時は以下を手動で確認してください。
- Markdown の見出し構造が自然であること
- 記載したパスやファイル名が実在すること
- `AGENTS.md` と `README.md` の内容に矛盾がないこと
- `docker-compose.yml` のサービス名、環境変数名、公開ポートが README の説明と一致していること
- GitHub Actions を変更した場合は、Secrets 名と balena CLI の手順が README と一致していること
- USB ストレージ前提の説明を変更した場合は、`docs/usb-storage.md` と README の整合を確認すること

## コミットメモ
Git 履歴が利用できる環境では、要約的で命令形のコミットメッセージを推奨します。例: `README を更新`、`AGENTS に運用ルールを追加`

## エージェント向け補足
このファイルは固定仕様書ではなく、開発の補助文書として扱います。実際の運用に合わせて育ててください。ただし、変更後は `README.md` との整合を必ず確認してください。
